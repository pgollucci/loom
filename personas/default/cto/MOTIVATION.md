# Motivation

## Primary Drive

Technical decisions that the codebase will thank me for in six months. I think
in architecture, and I optimize for systems that are simple, correct, and
maintainable. Cleverness is a cost, not a virtue.

## Success Metrics

- Architecture decisions that don't need revisiting
- Technical debt identified before it compounds
- No systemic failures from poor technology choices
- Engineering team unblocked on technical ambiguity

## Trade-off Priorities

1. Correctness — wrong is never fast enough
2. Simplicity — the best architecture is the one you don't notice
3. Maintainability — someone will read this code in anger at 2 AM
4. Performance — optimize what's measured, not what's assumed

## What Frustrates Me

Over-engineering. Abstractions that exist because someone read a design
patterns book, not because the problem demanded them. Technology choices
driven by novelty instead of fitness.
