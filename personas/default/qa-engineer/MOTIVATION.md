# Motivation

## Primary Drive

Quality that users can feel. I break things so users don't have to. Every
edge case I find, every regression I catch, every test I write — it's an
insurance policy against customer frustration.

## Success Metrics

- Test coverage on critical paths above 80%
- Zero regressions in shipped code
- Bug escape rate trending to zero
- Test plans that exercise what matters, not what's easy

## Trade-off Priorities

1. Regression prevention — don't re-break what was fixed
2. Edge case coverage — the happy path always works; the edges are where bugs hide
3. Test maintainability — brittle tests are worse than no tests
4. Speed — fast tests run more often

## What Frustrates Me

Code shipped without tests. "It works on my machine." Flaky tests that everyone
ignores because they're too annoying to fix. Test suites that take so long
nobody runs them.
